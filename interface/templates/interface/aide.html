Bienvenue sur la page d'aide de ruptures_interface

Le but de cette page est de vous aider à prendre en main l'outil. Pour cela, on va passer en revue les différentes pages de l'outil et comment les utiliser.

Accueil

Sur cette page, vous avez la possibilité de téléverser vos signaux sous la forme de fichier csv. Ci-dessous, se trouvent quelques exemple de fichiers acceptés par cet outil. Il faut dans le cas de signaux à plusieurs dimensions, séparer les colonnes à l'aide d'un espace.

figure 1 : Voici l'exemple d'un fichier csv du nom 1.csv à une dimension qui a un header ('Valeur0')

figure 2 : Voici l'exemple d'un fichier csv à une dimension sans header

figure 3 : Voici l'exemple d'un fichier csv à plusieurs dimensions avec header

Outre les fichier csv, vous pouvez téléverser des fichiers json qui contiennent les indices des endroits où il y a des ruptures.

figure 4 : Capture d'écran du fichier 1.json qui contient les indices des ruptures du fichier de la figure 1

Le téléversement des fichiers json est optionnel, mais si vous le faites, il faut respecter une convention de nommage. Le fichier json contenant les labels du signal x.csv doit s'appeler x.json.

Label

Sur la page Label, vous avez la possibilité de labelliser vos signaux pour y indiquer les endroits où se produisent les ruptures sur le signal. Si à l'étape précédente, vous avez téléversé des fichiers json (en respectant la convention de nommage), les labels seront automatiquement tracés pour vous (vous aurez quand même la possibilité de les modifier). Voici la liste des actions qui peuvent être réalisées sur cette page :

  • poser un label : pour cela, il faut cliquer sur le graphique à l'endroit où vous voulez poser un label.
  • déplacer un label : une fois posé, vous pouvez déplacer un label en le sélectionnant (drag and drop).
  • supprimer un label : lorsque vous voulez supprimer un label, vous devez passer la souris sur la zone où se trouve la liste des labels posés (sur la droite) le label correspondant sera alors en bleu pour vous aider à le remarquer. Une croix rouge apparaîtra pour supprimer ce label.
  • Valider : À chaque fois que vous avez fini de poser les labels, vous devez valider ces derniers, une fois que vous avez labellisé tous les signaux vous devez cliquer le bouton "Entraîner le modèle".

Prédiction

Sur la page Prédiction, vous avez maintenant la possibilité de prédire les ruptures sur vos signaux. En effet, le but de l'étape précédente était d'entraîner le modèle à détecter les ruptures sur des signaux "spécifiques". Pour obtenir vos premières prédictions, vous devez téléverser dans l'espace prévu à cet effet vos signaux. Une fois tous vos signaux téléversés, vous pouvez appuyer sur le bouton prédire pour voir les résultats. Vous tomberez sur une page similaire à la page Label, où vous pourrez cliquer sur les fichiers et afficher les points de ruptures prédits par le module.

C'est optionnel, mais vous avez la possibilité de téléverser des fichiers json pour comparer la prédiction du module et des labels détecté 'à la main'. Pour cela, comme sur la page d'accueil, il y a une convention de nommage à respecter. Le fichier json contenant les labels du signal x.csv doit s'appeler x.true.json.
