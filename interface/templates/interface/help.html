Welcome to the Ruptures Interface help page

This page point is to help you to get started with the Ruptures tools. To do so, we're going to see each pages/functionality et and how to use theam

Home

On this page, you have the possibility to upload your signals files as csv. Below you can find some exemple of accepted files. For csv files with several columns, please separete them with a blank space.

Besides csv, files, you can upload json files that contain the position of ruptures in a signal.

figure 4 : Screenshot of the file 1.json that contains the position of the ruptures for the figure 1 signal

The json file upload is optional, but if you do so, you have to give a specific name for the file. The json file containing the labels for the signal x.csv must be named x.json.

Label

On the Label paeg, you can put labels to your signals to show where break occurs in the signal. If at the previous step, uploaded a json file (respecting the name), labels will be automatically added to the signals (you still have te possibility to delete or modify them). This list is all the possible action that can be performed in this page :

  • put a label: to do so, you just have to click on the signal where you want to add the label.
  • move a label : once there is a label, you can simply move it (drag and drop).
  • delete a label : If you want to delete a label, you just have to put the mouse above the label info (on the right side), it will colour the label in blue and a red cross will appear. Click on that cross to delete the label.
  • Confirm : Once you have finished labeling a signal, you have to confirm the labels. When you confirmed labels for all signals, you can click on the button "Train".

Prédiction

On the Préeiction,you have the possibility to detect breaks point on your signals. To get your first results, you have to upload your signals. Once you uploaded all you signals, you can click on the detect button to get the detected breaks in your signals.

The json file upload is optional, but if you do so, you have to give a specific name for the file. Doing so, you have the possibility to compare between breaks that you found and the ones found by Ruptures. The json file containing the labels for the signal x.csv must be named x.true.json.
